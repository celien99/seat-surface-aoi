# 硬件对接说明

## 对接边界

当前代码只提供接口和模拟驱动，不接入真实厂商 SDK。测试机联调时只替换硬件实现，不修改共享内存协议。

## C++ 负责范围

- PLC 触发输入、OK/NG/RECHECK 输出。
- 相机初始化、曝光/增益/触发模式配置、图像采集。
- 频闪光源通道参数、频闪序列和异常关闭。
- 图像包写入共享内存，等待 Python 检测结果。

Python 不允许控制 PLC、相机和频闪。

## 接口实现要求

- 相机实现 `CameraDevice` 的初始化、采集和健康状态。
- 光源实现 `LightController` 的通道配置、序列执行和关闭输出。
- PLC 实现 `PlcClient` 的触发读取和结果输出。
- 所有等待必须有 timeout，任何超时不能输出 `OK`。
- 设备异常、缺帧、光源失败、PLC 输出失败必须进入 `RECHECK`、`ERROR` 或停线策略。

## 测试机联调顺序

1. 只接相机，使用模拟光源/PLC，验证图像尺寸、stride、pixel_format、timestamp。
2. 接频闪控制器，验证每个 `light_index` 与实际通道一致。
3. 接 PLC 触发输入，不输出分拣信号，只打印结果。
4. 接 PLC 输出，先用低速节拍和人工确认。
5. 连续运行 8 小时，统计缺帧、超时、slot 泄漏和误判。
